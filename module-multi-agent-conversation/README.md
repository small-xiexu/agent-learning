# module-multi-agent-conversation

## 模块定位

`module-multi-agent-conversation` 用于承载 AutoGen 风格的“对话驱动协作”范式。  
它把复杂任务理解为多个专家 Agent 之间的群聊推进，而不是一个中心调度器对多个子任务的机械分发。

该模块强调的不是“顺序编排”，而是：

- 谁当前拥有控制权
- 何时把控制权移交给另一位专家
- 状态如何在交接中持续保留

## 🎯 最佳实践场景

**自动化软件开发群聊**：PM 拆解需求、程序员写代码、审查员找漏洞，当前角色遇到能力盲区时，主动将控制权（Handoff）交接给下一位专家。

## 理论背景

对话驱动协作范式的核心思想是：  
把复杂任务映射为多角色之间的连续交流。

相比中心化流程控制，它更像一个动态的专家协作系统：

- 当前专家先处理自己擅长的问题
- 当发现问题超出自己边界时，把任务交给别的专家
- 新专家接管后继续推进，而不是从头再来

这种模式的优势在于：

- 自然贴近专家协作过程
- 便于按角色拆分职责
- 适合开放式、多轮推进的复杂任务

## 运行机制

该模块建议采用“活动 Agent + 交接协议”的机制。

### 1. 当前活动 Agent

任意时刻只有一个 Agent 处于主导位置，负责直接响应当前用户或系统状态。

### 2. 控制权判断

活动 Agent 在每轮处理后都要判断：

- 我能否继续处理
- 我是否需要请求另一位专家接管
- 当前状态是否已经足够让下一位专家继续推进

### 3. Handoff

一旦触发交接，系统需要同时转移两样东西：

- 控制权
- 状态

只有转移控制权而不转移状态，会导致新专家“失忆”；  
只有转移状态而不切换活动 Agent，又无法实现真正接管。

### 4. 终止

当某位 Agent 判断任务已完成，或者系统策略判定无需再继续交接时，流程结束。

## Spring AI Alibaba 映射

这个模块与 Spring AI Alibaba 官方文档中的 **Handoffs（交接）模式** 高度对应。

官方文档明确指出，在交接模式中：

- 当前 Agent 可以决定需要另一位 Agent 的帮助
- 它会把控制权和状态传递给下一位 Agent
- 新 Agent 作为新的活动 Agent 与用户继续交互

这正是本模块的核心机制。

### 关键映射点

- `Handoffs`：对应控制权迁移
- `instruction` / `systemPrompt`：定义每位专家的职责边界与交接条件
- 状态与占位符：确保新 Agent 能读取前序结果而不是从零开始

### 工程上的一个重要判断

对话驱动协作不等于没有治理。  
对于开放式群聊，`Handoffs` 是主机制；  
对于需要更强控制的企业场景，可以在外围叠加 `SupervisorAgent` 或显式治理策略，但不应破坏“交接才是真正切换主导权”的核心思想。

## 与 framework-core 的关系

本模块需要把对话协作建立在统一协议之上：

- `Message / Memory`：保存多 Agent 共享的会话历史
- `AgentLlmGateway`：统一专家 Agent 的模型调用入口
- `ToolRegistry`：让专家不仅能聊天，还能调用能力

推荐做法是把“交接事件”也建模为一种显式领域事件，而不是只作为内部回调。

## 工程落地建议

### 1. 角色边界必须清晰

多专家协作失败，往往不是模型不够强，而是多个 Agent 职责重叠，导致反复拉扯。

### 2. 交接条件必须显式

建议明确约束：

- 什么情况下必须交接
- 什么情况下禁止交接
- 什么情况下交接后必须保留哪些状态

### 3. 控制群聊长度

去中心化对话最怕无止境接力。  
应设置最大交接次数、最大回合数和终止策略。

### 4. 专家接管不等于全量上下文暴露

即使交接，也不意味着必须暴露全部上下文。  
可根据任务需要过滤、裁剪或结构化传递状态。

## 适用场景与边界

### 适用场景

- 专家角色明显、边界清晰的多轮任务
- 需要动态接管的开放式复杂问题
- 对话式协作、咨询式协作、分析式协作

### 不适合的场景

- 必须严格按固定顺序执行的任务
- 状态机非常强、分支固定的工作流
- 合规要求极高、必须由中心节点审核每一步的流程

## 结论

对话驱动协作的本质不是“多几个 Agent 一起聊天”，而是把控制权迁移做成一等机制。

在本项目中，`module-multi-agent-conversation` 应承担去中心化协作范式的落地点，为专家接管、群聊推进和任务移交提供清晰的工程实现。
